Ref.EXP-01

Moved the OS job scheduler onto an FPGA. Throughput doubled.

  • 01Moved scheduling off the CPU and onto a Xilinx Alveo U55C FPGA. With the FPGA owning the scheduler, the host CPU stopped burning cycles managing the work it could be doing.
  • 02Removed the host–FPGA double-copy by sharing physical memory between them; throughput doubled once those redundant transfers were gone.
  • 03Built a three-thread C++ pipeline — Writer, Reader, Logger — keeping two FPGA kernels fed continuously: lock-free ring buffers on the hot path, mutexes elsewhere.
  • 04Moved the task queue into hardware so high-priority jobs bypass the OS scheduler, dropping latency to microseconds.
