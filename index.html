Ref.EXP-01

Pulled the OS job scheduler off the CPU and onto an FPGA. Throughput doubled.

  • 01Put the scheduler on a Xilinx Alveo U55C. Once the FPGA owned scheduling, the host CPU stopped burning cycles babysitting work it should have just been running.
  • 02The host and FPGA were copying the same data across the bus twice. Shared their physical memory instead, killed the redundant transfers, and throughput doubled.
  • 03Wrote a three-thread C++ pipeline — Writer, Reader, Logger — to keep two FPGA kernels fed without stalling. Lock-free ring buffers on the hot path; mutexes only where contention didn't matter.
  • 04Moved the task queue into hardware so urgent jobs skip the OS scheduler entirely. Latency dropped to microseconds.
